# Logic Generator

Compile a human-written **formula** into an **Approximately Up Demo** in-game
logic *blueprint*. You get a live circuit visualization in the browser and a
game-importable blueprint bundle (`.bp` + `.bpmeta` [+ `.bpex`]) as a downloadable
ZIP.

- **Stack:** React + Vite + TypeScript, npm.
- **Version:** `0.1.3` (bump on every change — see `package.json`).

```
Formula ──► lexer ──► parser (AST) ──► compiler (block graph, CSE) ──►
        layout (tidy 2D grid + cable routing) ──► ┬─► SVG visualization
                                                  └─► serializer ─► ZIP (.bp/.bpmeta/.bpex)
```

## Run it

```bash
cd Logic-Generator
npm install
npm run dev        # http://localhost:5173
npm run build      # type-check (tsc -b) + production bundle to dist/
npm run preview    # serve the production build
npm test           # vitest unit tests (lexer, parser, compiler, _gt, serializer)
```

## The formula language

Each line assigns a **named signal**: `name = expression`.

- **Free variables** (used but never assigned) become **Input** blocks.
- **Assigned variables that nothing else consumes** become **Output** blocks.
- **Assigned variables used downstream** are shared internal wires (fan-out).

```text
error   = target - position
control = Kp * error + Kd * deriv(error)
```

`target`, `position`, `Kp`, `Kd` → inputs; `control` → output; `error` → shared wire.

### Operators (exactly what the game's logic blocks provide — nothing more)

| Kind | Spelling |
|---|---|
| arithmetic | `+ - * /`, `min(a,b) max(a,b) mod(a,b) pow(a,b)` (or `a^b`) |
| math | `abs sqrt exp log` |
| trig | `sin cos tan asin acos atan atan2(y,x) sinh cosh tanh asinh acosh atanh` |
| logic | `not(a) xor(a,b) condition(a,b)` |
| stateful | `deriv(x)`/`d(x)`, `integral(x)`/`integ(x)`, `memory(x)` |
| shaping | `remap(x,inMin,inMax,outMin,outMax)`, `threshold(x,t)` |

Precedence: `^` (right-assoc) > unary `-` > `* /` > `+ -`. Parentheses group.
Anything outside this catalog (unknown functions, wrong arity) is rejected with a
clear error message.

## Blueprint format (what is implemented)

- **`.bpmeta`** — ✅ complete. UTF-8 JSON `{"_name","_folder","_version":"0.1.139"}`.
- **`.bp`** — header + payload records.
  - **Header:** ✅ reused **verbatim** from a real v0.1.139 reference file
    (`data/989e5da9… PD Target Distance.bp`, first 3220 bytes = 36 structs),
    so it is guaranteed valid. Regenerate with `node scripts/extractHeader.mjs`.
  - **Records:** ✅ correct grammar
    `SCPrefab(u64) · structIndex(i32) · BlueprintData(sizeof) · trailing(i32)`,
    tightly packed to EOF. Field values (`_guid`,`_gt`,`_value`,`_col`) are written
    at the offsets read from the header (located by field-name hash).
  - **`_gt`:** ✅ packed with the disassembly-proven layout
    `X | Y<<9 | Z<<18 | rot<<27` (cell = 0.125), from GT_REPORT. Default rotation is
    the **upright** index `0` (real blocks use it; identity index 16 renders sideways)
    and the circuit is offset into the **interior** of the grid (`INTERIOR_BASE_CELL`)
    instead of the (0,0,0) corner.
- **`.bpex`** — ✅ emitted as 192 KiB of zeros (toggleable).
- **ZIP** — ✅ bundled with `fflate` and downloaded from the browser.

## ⚠ Serializer pieces that are STILL STUBBED / PROVISIONAL

These are isolated in clearly-marked config so they can be finalized without
touching the rest of the app. Until then the exported ZIP is **structurally valid
and parses to the exact byte**, but is *best-effort* for in-game correctness.

1. **Operator → SCPrefab hashes** (`src/serializer/prefabTable.ts`).
   The name→hash algorithm `AsciiHash64` is uncracked (see `scratch/REPORT.md §5`).
   - **Confirmed:** `Constant`, `Differentiator`, `Accumulator`, `Cable`.
   - **Placeholder (TODO):** every other operator (Adder, Subtractor, Multiplier,
     Divider, trig, logic, routing, Remapper, Threshold, Memory, and the
     input/output terminals). They currently reuse a valid generic signal-processor
     identity so the file loads; the block *type* will be wrong until the real hash
     is supplied. The UI flags exactly which ops are provisional per compile.
2. **I/O terminal prefabs** — the real "named input/output" block prefabs are not
   yet identified (using the generic placeholder).
3. **Cable geometry** (`bpWriter.ts`, `emitCables`). We emit one cable-cell record
   per routed grid cell, but `_shape`/`_type` (the connectivity/orientation enum)
   are unknown (`scratch/REPORT.md §8`), so they are written as `0`. Cables are
   therefore positional placeholders, not wired connectivity. Toggle off if the
   loader rejects zero-shape cables.
4. **`_gt` open question.** The layout above is proven from the game's Encode/Decode
   machine code (`GT_REPORT.md`). A separate empirical pass observed on-disk `_gt`
   values that look bit-diffused (`REPORT.md §7`). If in-game tests confirm an extra
   reversible permutation, wrap it inside `packGt`/`unpackGt` in `gtCodec.ts` — no
   other module changes.

**To finalize:** crack `AsciiHash64` (or dump a few known hashes in-game), fill the
real hashes into `prefabTable.ts`, add the `_shape` enum table for cables, and
confirm `_gt` — the serializer then produces fully correct blueprints unchanged.

## Changelog

### 0.1.1
- **Fix (rotation):** default block rotation changed from the identity index `16`
  to the upright index `0` (`ROT_UPRIGHT`). Real authored blocks store `rot=0`
  (GT_REPORT.md §5, PD Target Distance record 0); the identity quaternion rendered
  blocks "on their side" in-game.
- **Fix (position):** the whole generated circuit is now translated into the
  interior of the 0..511 grid volume via the named `INTERIOR_BASE_CELL` constant
  (`layout.ts`, default `{200,200,200}`) instead of starting at the (0,0,0) corner,
  which had put every block in one corner of the build area.
- Tests: updated the default-rotation assertion and added `layout.test.ts`
  (default rotation = 0, all cells land in the interior region and inside 0..511).
- ⚠ **Open / needs in-game confirmation:** the shipped-blueprint `_gt` values are
  bit-diffused (verified over 12k samples via `scratch/decodeRegion.js`), so the
  exact interior region, the up-axis (which of X/Y/Z is vertical), and any per-block
  rotation could not be read back from the references. The upright index and mid-
  volume offset are the best-supported defaults pending an in-game test.
