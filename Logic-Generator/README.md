# Logic Generator

Compile a human-written **formula** into an **Approximately Up Demo** in-game
logic *blueprint*. You get a live circuit visualization in the browser and a
game-importable blueprint bundle (`.bp` + `.bpmeta` [+ `.bpex`]) as a downloadable
ZIP.

- **Stack:** React + Vite + TypeScript, npm.
- **Version:** `0.1.7` (bump on every change ‚Äî see `package.json`).

## The formula language

Each line assigns a **named signal**: `name = expression`.

- **Free variables** (used but never assigned) become **Input** blocks.
- **Assigned variables that nothing else consumes** become **Output** blocks.
- **Assigned variables used downstream** are shared internal wires (fan-out).

```text
error   = target - position
control = Kp * error + Kd * deriv(error)
```

`target`, `position`, `Kp`, `Kd` ‚Üí inputs; `control` ‚Üí output; `error` ‚Üí shared wire.

### Operators (exactly what the game's logic blocks provide ‚Äî nothing more)

| Kind | Spelling |
|---|---|
| arithmetic | `+ - * /`, `min(a,b) max(a,b) mod(a,b) pow(a,b)` (or `a^b`) |
| math | `abs sqrt exp log` |
| trig | `sin cos tan asin acos atan atan2(y,x) sinh cosh tanh asinh acosh atanh` |
| logic | `not(a) xor(a,b) condition(a,b)` |
| stateful | `deriv(x)`/`d(x)`, `integral(x)`/`integ(x)`, `memory(x)` |
| shaping | `remap(x,inMin,inMax,outMin,outMax)`, `threshold(x,t)` |

Precedence: `^` (right-assoc) > unary `-` > `* /` > `+ -`. Parentheses group.
Anything outside this catalog (unknown functions, wrong arity) is rejected with a
clear error message.

## Blueprint format (what is implemented)

- **`.bpmeta`** ‚Äî ‚úÖ complete. UTF-8 JSON `{"_name","_folder","_version":"0.1.139"}`.
- **`.bp`** ‚Äî header + payload records.
  - **Header:** ‚úÖ reused **verbatim** from a real v0.1.139 reference file
    (`data/989e5da9‚Ä¶ PD Target Distance.bp`, first 3220 bytes = 36 structs),
    so it is guaranteed valid. Regenerate with `node scripts/extractHeader.mjs`.
  - **Records:** ‚úÖ correct grammar
    `SCPrefab(u64) ¬∑ structIndex(i32) ¬∑ BlueprintData(sizeof) ¬∑ trailing(i32)`,
    tightly packed to EOF. Field values (`_guid`,`_gt`,`_value`,`_col`) are written
    at the offsets read from the header (located by field-name hash).
  - **`_gt`:** ‚úÖ packed with the disassembly-proven layout
    `X | Y<<9 | Z<<18 | rot<<27` (cell = 0.125), from GT_REPORT. Default rotation is
    the **upright** index `0` (real blocks use it; identity index 16 renders sideways)
    and the circuit is offset into the **interior** of the grid (`INTERIOR_BASE_CELL`)
    instead of the (0,0,0) corner.
- **`.bpex`** ‚Äî ‚úÖ emitted as 192 KiB of zeros (toggleable).
- **ZIP** ‚Äî ‚úÖ bundled with `fflate` and downloaded from the browser.

## ‚öÝ Serializer pieces that are STILL STUBBED / PROVISIONAL

These are isolated in clearly-marked config so they can be finalized without
touching the rest of the app. Until then the exported ZIP is **structurally valid
and parses to the exact byte**, but is *best-effort* for in-game correctness.

1. **Operator ‚Üí SCPrefab hashes** (`src/serializer/prefabTable.ts`).
   `AsciiHash64` is uncracked ‚Äî hashes must be **read from real `.bp` files**, not computed
   from names (see *Prefab hash extraction* below).
   - **Confirmed (v0.1.6, Block List blueprint):** all 19 formula OpKeys present in the
     reference row plus 9 extra blocks (Logic Value, Addition Array, Joystick Splitter,
     six sensor meters). Full table: `data/prefab-map.json`.
   - **Fixed vs v0.1.5:** Divider (`0x00766e‚Ä¶`, was Addition Array `0x3b80bd‚Ä¶`),
     Router4 (`0x0124da‚Ä¶`, was `0x1fa8fb‚Ä¶`).
   - **Placeholder (TODO):** trig, `condition`, and other ops not in the Block List row.
2. **I/O terminal prefabs** ‚Äî wireless transmitter hash confirmed; channel assignment is heuristic.
3. **Cable geometry** ‚Äî per-cell `_gt.rot` from connectivity map; `_shape` still `0` in some cases.
4. **`_gt`** ‚Äî plain pack `X|Y<<9|Z<<18|rot<<27` (GT_REPORT_v2.md). Default block rot is
   `ROT_LOGIC` (index **3**, 180¬∞ Y flip from reference upright index 6).

### Prefab hash extraction (can hashes be READ instead of computed?)

**Yes ‚Äî reading from real blueprints is the reliable method.** Computing from
`SC_Adder`-style strings failed (`scratch/crackHash*.js`, `scratch/REPORT.md ¬ß5`).

1. **Parse `.bp` payload** ‚Äî each record is `prefab(8) | idx(4) | entityId(20) | _gt(4) | ‚Ä¶ | trailing(4)`.
   Signal processors use `idx=0`; the 8-byte `prefab` ulong is the block identity.
2. **List distinct idx=0 hashes** ‚Äî from the repo root:
   `node scratch/analyzeEntities.js` (uses `scratch/entities_*.json`), or
   `node scratch/walkPayload.js path/to/file.bp` to dump one file.
3. **Label by topology** ‚Äî match prefab counts to known circuit structure (e.g. PD Target
   Compact `feca9562‚Ä¶`: 2√ó mul `0x0407‚Ä¶`, 1√ó sub `0x1c97‚Ä¶`, 1√ó router2 `0x5929‚Ä¶`, 1√ó add
   `0x3b7b‚Ä¶`). Cross-check display names in `scratch/game-mined/Localization.csv`
   (`SC_<Name>_Name` keys in `prefab_names_SCkeys.txt`).
4. **Alternatives:** runtime dump via BepInEx calling `BurstUtility.AsciiHash64`, or
   AssetRipper prefab inspection ‚Äî still empirical, not a closed-form formula.
